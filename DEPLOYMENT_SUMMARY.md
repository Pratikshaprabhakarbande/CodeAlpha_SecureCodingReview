## 🎤 INTERVIEW PREPARATION - 30-60 SECOND PITCH

### The Pitch
"I created and deployed 'Secure Coding Review' - a GitHub project demonstrating real-world security vulnerabilities. 

The application contains 9 intentional security issues including SQL injection, command injection, and weak cryptography. I analyzed the code with Bandit static analyzer and created comprehensive documentation explaining each vulnerability, how to exploit it, and how to fix it.

This project shows I understand both how vulnerabilities occur and how to prevent them. The professional GitHub setup demonstrates version control expertise, and the detailed documentation shows I can communicate technical concepts clearly."

### Questions to Prepare For
**Q1: "Why did you create this project?"**
A: "To understand security vulnerabilities from a hacker's perspective. By building vulnerable code intentionally, I learned how real attacks work and how to write secure code that prevents them."

**Q2: "Which vulnerability is the most critical?"**
A: "SQL Injection or Command Injection - both allow complete system compromise. SQL injection can bypass authentication entirely, while command injection enables arbitrary code execution."

**Q3: "How would you exploit the SQL injection?"**
A: "Username: admin' -- bypasses the password check. The comment removes the password verification, allowing login as admin without a password."

**Q4: "What's wrong with MD5 for passwords?"**
A: "MD5 is cryptographically broken, fast to compute (enabling brute force), has known collisions, and rainbow tables exist for common passwords. Bcrypt or Argon2 are much better choices."

**Q5: "How does Bandit help?"**
A: "It's a static security analyzer that scans Python code for common vulnerabilities automatically. It saved me time finding issues and helped me understand different vulnerability classes."
